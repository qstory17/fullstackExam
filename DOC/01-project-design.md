프로젝트 설계

1. 서비스 설명

"나의 개발 다짐 게시판"
닉네임과 개발 목표를 본인, 타인이 서로 공유할 수 있는 방명록 서비스입니다.

2. 전체 흐름

   1)사용자 : 웹 브라우저에서 닉네임과 내용을 입력하고 '등록' 버튼을 클릭합니다. 2)프론트엔드: 입력된 데이터를 JSON 형식으로 만들어 백엔드 API(POST /api/guestbooks)로 전송합니다. 3)백엔드: 요청을 받아 데이터를 검증하고, MySQL 데이터베이스에 영구 저장합니다. 4)프론트엔드: 저장 성공 응답을 받으면, 다시 백엔드에 목록 조회(GET /api/guestbooks)를 요청하여 최신 데이터를 화면에 보여줍니다.

3. 프론트엔드 역할 (Next.js)

- 사용자가 볼 수 있는 UI(입력 폼, 리스트)를 제공합니다.
- fetch API를 사용하여 백엔드와 비동기 통신을 합니다.
- 데이터를 화면에 즉각적으로 반영(Rendering)하여 사용자 경험을 높입니다.

4. 백엔드 역할 (Spring Boot)

- 프론트엔드의 요청을 받는 창구(REST API Controller) 역할을 합니다.
- 비즈니스 로직(데이터 가공, 검증 등)을 처리합니다.
- JPA(Repository)를 사용하여 데이터베이스(MySQL)와 직접 소통하고 데이터를 관리합니다.
- CORS 설정을 통해 다른 도메인(프론트엔드)의 접근을 허용합니다.

5. 배포 구조 요약

- 로컬 환경: 내 컴퓨터 안에서 localhost:3000(프론트)과 localhost:8080(백엔드)이 통신합니다. DB도 내 컴퓨터에 설치된 것을 사용합니다.
- 배포 환경 (예정 - 미구현): AWS EC2 서버 한 곳에 Docker 컨테이너 3개(Frontend, Backend, DB)를 띄워서 서로 통신하게 만듭니다. 사용자는 EC2의 공인 IP로 접속하게 됩니다.
